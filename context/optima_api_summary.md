# Выжимка: Оптима API × Жемчужина

> Источники: PDF руководства Optima Business API v1.0 (12.06.2026) + голосовое Марсель 12.06.2026
> Зафиксировано: 2026-06-13

## Как платит родитель

| Способ | Комиссия | Как |
|---|---|---|
| Optima 24 (приложение) | Бесплатно | "Образование" → Жемчужина → PIN ребёнка → сумма |
| Терминал Оптима | 30 сом с родителя | Наличные в терминале |

## API возможности (из PDF)

**Базовый URL:** `https://api.optimabusiness.kg/`
**Аутентификация:** заголовок `X-API-KEY: ключ`
**Ключ генерирует:** первый подписант в кабинете Optima Business

| Функция | Эндпоинт | Метод |
|---|---|---|
| Генерация QR | `/api/v2/generate/qr` | POST |
| Статус транзакции по QR | `/api/v1/get-qr-transaction-info/{id}` | GET |
| Выписка по QR | `/api/v1/get-qr-operation-statement` | GET |
| Остатки по счетам | `/api/v1/get-account-balances` | GET |

## Callback (real-time)

Оптима сама POST'ит на наш сервер при каждой оплате:
- Наш URL: `https://наш_домен/api/v1/callback`
- Аутентификация входящего от банка: **Basic Auth** (мы даём логин+пароль)
- Тело callback:
```json
{
  "status": "PROCESSED",
  "sum": 18000.00,
  "note": "...",
  "transactionProcessedDateTime": "2026-06-13T10:00:00Z",
  "transactionId": "abc123",
  "payerClientType": ""
}
```

**Важно:** это НЕ D+1 — приходит мгновенно. D+1 — только API выписки.

## Наша модель интеграции (PIN)

Работаем через PIN, не QR:
- Родитель вводит PIN ребёнка в приложении → платит
- Оптима шлёт callback на наш сервер
- Мы по идентификатору в callback находим студента → помечаем оплату
