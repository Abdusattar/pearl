# ИС Жемчужина — полный список модулей (2026-05-31)

> Приоритеты (MVP / потом) — уточнить с Абдусаттаром отдельно

## Дети / семьи
- Карточка ребёнка (ФИО, PIN, класс/группа, контакты родителей)
- Посещаемость (школа + садик)
- Питание (меню, учёт по дням)
- Развоз (маршруты, кто едет, расписание)
- Медицина (аллергии, группа здоровья, справки)

## Финансы
- Оплаты (Optima/Бакай → выписка → привязка к ребёнку)
- Расходы (фото квитанций → OCR → категории)
- Зарплаты сотрудников / табель
- Отчёты для владельцев

## Учебный процесс
- Учебные планы
- Успеваемость / оценки
- Домашние задания
- Расписание уроков/занятий
- Секции и кружки

## CRM / Коммуникация
- CRM (набор, лиды, родители)
- Уведомления родителям (через Telegram бот)
- Объявления по классам/группам
- Договоры с родителями

## Операционка
- Сотрудники (карточка, роль, зарплата)
- Инвентарь / имущество (мебель, техника, учебники)
- Поставщики (еда, товары)
- Мероприятия (праздники, утренники)

## Отчётность
- Отчёты для владельцев (финансовая картина)
- Госотчётность (если лицензированная школа)

## Технологический стек (согласовать с мудрецом)

**Клиенты:**
| Кто | Через что | Что делает |
|---|---|---|
| Владельцы / управляющие | Веб | Отчёты, картина по деньгам |
| Сотрудники (учителя, Мунара) | Веб + Telegram бот | Ввод данных, фото квитанций |
| Родители | Telegram бот | QR-коды, справки, уведомления |
| Абдусаттар (разраб) | Терминал + Claude Code | Строит, меняет, контролирует |

**Стек (финальный, согласован с мудрецом 31.05.2026):**
- Backend: FastAPI (Python) — веб + бот в одном процессе
- БД: PostgreSQL + JSONB для гибких полей
- Frontend: HTMX + Jinja2 — простые таблицы/формы, без npm
- Бот: python-telegram-bot как FastAPI endpoint (не отдельный сервис)
- OCR квитанций: Google Cloud Vision (1000 запросов/мес бесплатно)
- Деплой: Docker + VPS (облако)
- Auth: Session cookies (не JWT)
- Multi-tenant: shared schema с organization_id (школа + садик)
- Audit log: таблица изменений (кто/что/когда)

**Философия:** Живой слепок бизнеса. Система меняется вместе с бизнесом — гибко, без переписывания. Не монолит. Два клиента (веб + бот) к одному backend.
