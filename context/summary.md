## Логика авто-склада (важно)

При подтверждении квитанции:
- Категория `warehouse_eligible = true` (только «Продукты питания») → показывается зелёный чекбокс «Добавить в склад»
- По умолчанию включён. При Провести → `WarehouseReceipt` создаётся для каждой позиции с qty и price
- Категория «Готовая еда» → `warehouse_eligible = false` → чекбокс не показывается → идёт прямо в расход (счёт 7510)
- `WarehouseReceipt.transaction_id` связывает складской приход с расходной транзакцией

## Бэклог — следующие модули (приоритет)

### 1. Поставщики (Suppliers) — простой, делать первым
- Справочник: название + телефон (необязательный)
- Телефон уникальный если указан
- При покупке на рынке/в магазине → выбирают "Рынок" или "Магазин Народный" (без телефона)
- Квитанция привязывается к поставщику → история закупов по каждому

### 2. Оплата поставщикам (Кредиторская задолженность) — средний
- У каждого закупа статус: `оплачено сразу` / `оплата позже` / `погашено`
- Дата планируемой оплаты
- Итог: видно сколько школа/садик должна поставщикам сейчас
- Актуально для летних периодов (берут в долг, платят позже)

### 3. Меню → Склад (Расход продуктов) — сложный, большой проект
- На каждый день/приём пищи (завтрак/обед/ужин) создаётся меню
- Каждое блюдо → рецепт с граммовками ингредиентов
- При проведении меню → автоматически списываются продукты со склада
- Пример: "Плов 30 детей" → рис −3кг, морковь −1.5кг, масло −0.5л
