> Создано: 2026-06-13, обновлено: 2026-07-03
> Источники: PDF руководства Optima Business API v1.0, протокол «Интерфейс подключения
> поставщиков услуг» (QIWI/OSMP, вер. 1.4, `банки/оптима/протокол (11).pdf`)

## Закрыто 03.07

### 1. Структура интеграции для Education/PIN платежей — РЕШЕНО
Это не QR-callback, а классический OSMP-протокол «check+pay»: банк сам шлёт
GET-запросы на наш эндпоинт `/optima/payment` (см. `app/routers/optima.py`).
- `command=check` — банк спрашивает, существует ли PIN, мы отвечаем ФИО+группой
- `command=pay` — банк подтверждает платёж, мы фиксируем `Transaction`
- Реализовано и задеплоено, банк начал реальное тестирование 03.07

### 2. D+1 vs real-time — РЕШЕНО
Real-time (check+pay). D+1 — только для ежедневной сверки (реестр по email от банка,
до 10:00 МСК). Email для приёма сверки пока не согласован с банком отдельно.

### 5. PIN-список — обновлено
Формат сменился с 3 на 4 знака (0001–9999, без переиспользования) 03.07 — пока
родители не пользовались боевыми PIN, миграция бесплатна. Банку переданы
тестовые фиктивные PIN 9001/9002 (не реальные дети) для тестовых платежей.
**Актуальный PIN-список садика Сокулук нужно переотправить банку** — старый
файл с 3-значными кодами больше не соответствует системе.

### 6. HTTPS endpoint — РЕШЕНО
`https://pearl-production-eef5.up.railway.app/optima/payment` — прод на Railway,
уже HTTPS.

## Ещё открыто

### A. Показывает ли приложение Optima родителю ответ из `check`?
Наш `check` возвращает `<comment>ФИО — Группа</comment>`. По описанию Абдусаттара
(разговор с банком) — похоже, что да, родитель видит имя перед подтверждением
платежа, это и есть страховка от ошибки с PIN. **Уточнить явно у Амана/Марселя.**

### B. IP-подсеть банка
В протоколе (шаблонном, ещё под брендом QIWI) указана 79.142.16.0/20 — это
исторически московский диапазон QIWI, вероятно неактуален для Optima КР.
Фильтрация по IP пока не настроена. Уточнить у банка реальный диапазон,
когда дойдёт до продакшена.

### C. Один эндпоинт на 3 услуги (Школа / Садик Сокулук / Садик Кожомкул)
Банк подтвердил — один эндпоинт. Работает, потому что PIN уникален по всей
организации — какую бы «услугу» родитель ни выбрал в меню банка, для нас это
не имеет значения, находим ребёнка просто по PIN.

### D. API ключ Optima Business (для другого — QR/выписка функционала)
Не обязателен для текущей check+pay интеграции, но остаётся открытым, если
понадобится API выписки/остатков отдельно.
