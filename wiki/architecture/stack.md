# Технологический стек — Жемчужина

> Финально согласован: 2026-05-31

## Стек

| Слой | Технология | Решение |
|---|---|---|
| Backend | FastAPI (Python) | Веб + Telegram бот в одном процессе |
| БД | PostgreSQL + JSONB | Гибкие поля без миграций для переменных атрибутов |
| Frontend | HTMX + Jinja2 | Простые таблицы/формы, **без npm и сборки** |
| Бот | python-telegram-bot | Как FastAPI endpoint, не отдельный сервис |
| OCR | OpenRouter Vision | Заменил EasyOCR (см. wiki/ocr/decision.md) |
| Деплой | Docker + VPS | Облако, один контейнер |
| Auth | Session cookies | Не JWT — проще для данного масштаба |
| Multi-tenant | Shared schema + `organization_id` | Школа + садик в одной БД |
| Audit | Таблица изменений | Кто/что/когда — требование прозрачности |

## Архитектурные принципы

- **Живой слепок бизнеса** — система меняется вместе с бизнесом, без переписывания
- **Два клиента к одному backend** — веб + Telegram бот через один API
- **Не монолит** — модули независимы, добавляются постепенно

## Почему именно этот стек

- FastAPI: быстро, async, отлично подходит под Python + Telegram
- HTMX: нет frontend-команды, нет npm — только server-rendered HTML с реактивностью
- PostgreSQL JSONB: гибкость для школьных данных без постоянных миграций
- Docker: один файл деплоя, воспроизводимость среды
- Session cookies: целевая аудитория — сотрудники школы, не публичный API

## Клиентская матрица

| Кто | Через что | Что делает |
|---|---|---|
| Владельцы / управляющие | Веб | Отчёты, картина по деньгам |
| Мунара (садик) | Веб + Telegram бот | Ввод данных, фото квитанций |
| Айжан (директор школы) | Веб | Отчёты, контроль |
| Родители | Telegram бот | QR-коды, уведомления |
| Абдусаттар (разраб) | Терминал + Claude Code | Строит и меняет |
