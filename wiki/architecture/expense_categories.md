# Категории расходов — Архитектура и решения

> Зафиксировано: 18.06.2026
> Источник: ответ NotebookLM CFO + диалог с Абдусаттаром

## Структура дерева (двухуровневая)

| Родительская | Дочерняя (листовая) | warehouse_eligible | Счёт КР (ориентир) |
|---|---|---|---|
| Питание | Продукты питания | ✅ true | 2160 → 7510 при списании |
| Питание | Готовая еда | false | 7510 (прямо) |
| Питание | Питьевая вода | false | 7510 |
| Хозяйство | Бытовая химия | false | 7160 |
| Хозяйство | Канцтовары | false | 7160 |
| Транспорт | — (листовая) | false | 7630 |
| Ремонт | — (листовая) | false | 7160 |
| Прочее | — (листовая) | false | 7810 |

**Не в Pearl (это 1С):** ФОТ, Соцфонд, Амортизация — вводит бухгалтер, не оператор.

## Ключевое разделение: Продукты vs Готовая еда

В 1С обрабатываются принципиально по-разному:

- **Продукты питания (сырьё):** поступают на склад (счёт 2160/2110). Списываются в расход (7510) только через меню-требование. Pearl хранит физические остатки в кг.
- **Готовая еда (услуга/кейтеринг):** закупленные готовые порции не хранятся. В 1С идут сразу в дебет 7510, минуя складские счета.

Именно поэтому флаг `warehouse_eligible` стоит только на «Продукты питания», но не на «Готовая еда».

## Флаг warehouse_eligible

Поле на модели `ExpenseCategory`. Управляет появлением чекбокса «Добавить в склад» на странице confirm.html.

**Почему флаг, а не substring-match по имени:**
- `name.contains("питан")` поймал бы и «Готовую еду» (дочерняя «Питания»)
- Явный флаг позволяет точно контролировать без зависимости от названия
- Будущие категории (например, «Молочные продукты») получат флаг при создании

## Что ещё рекомендовал финэксперт (не реализовано)

- Флаг «Есть ЭСФ (счёт-фактура)?» на расходе — без ЭСФ расходы не принимаются к вычету по НК КР (ст. 195)
- `supplier_inn` в Transaction — для проверки вычета НДС
- Категория «Медицинские расходы» — аптечки, медкнижки персонала (7160)
- Категория «Обучение персонала» — аттестация учителей (7310)
- Категория «Банковские комиссии» — комиссии Optima (7810)

Коды счетов (7510, 7160 и т.д.) — из типового плана счетов КР, не из конкретной 1С Жемчужины. Реальные «Статьи затрат» подтверждает бухгалтер по своей базе.
