## Доступ к системе (согласовано 02.06.2026)

| Кто | Доступ |
|---|---|
| Айжан (директор школы) | Школьные расходы — полный. Садики — только просмотр |
| Мунара (садики) | Расходы садиков, ввод платежей |
| Педагоги | Только свой класс, финансов нет |
| Уполномоченный (при необходимости) | Школьные расходы — ввод |
| Сотрудник закупа одного объекта (напр. Махабат) | Закуп/квитанции только своей организации (`role=staff`, `organization_id` фиксирован) — паттерн для любого будущего узкого доступа |

**Роль в БД (06.07):** enum-подобное поле `User.role` = `owner\|director\|manager\|staff`.
Раньше последнее значение называлось `teacher` — переименовано, т.к. название вводило
в заблуждение (сотрудник закупа — не педагог). Функционально `staff` (как и было
`teacher`) означает «не owner/director/manager» → доступ только к своей `organization_id`,
без доступа к другим объектам.

- Создание групп (классов/садиков): Мунара для садиков, Айжан для школы
- Расходы школы: вводит Айжан (+ уполномоченный при необходимости)

## Масштаб объекта

- **Школа:** ~19 классов × ~22 ребёнка = ~418 учеников (классы 0–4)
- **Садик:** 2 локации — Сокулук (основной, ~70 мест) + Кожомкул (~15 км ближе к Бишкеку)
- **Активных пользователей ИС:** ~50
- **Оплата:** ~13 000 сом/месяц за ребёнка (школа)

## Как принимаются решения

- **Архитектурные решения** — Абдусаттар предлагает, согласует с "мудрецом" (Claude)
- **Операционные решения** (роли, доступ, формат данных) — Айжан + Мунара подтверждают в Telegram
- **Банковская интеграция** — через Айжан (она инициирует контакт с банком)
- **Ритм:** каждый месяц — конкретный видимый результат

Pro-bono консалтинг. Оплата не взимается.
Абдусаттар — куратор и разработчик со стороны консультанта.
