## Как платит родитель

| Способ | Комиссия | Путь |
|---|---|---|
| Optima 24 (приложение) | Бесплатно | "Образование" → Жемчужина → PIN ребёнка → сумма |
| Терминал Оптима | 30 сом с родителя | Наличные |

## API — что задокументировано

**Базовый URL:** `https://api.optimabusiness.kg/`
**Аутентификация:** заголовок `X-API-KEY`
**Ключ:** генерирует первый подписант в кабинете Optima Business

| Функция | Эндпоинт | Метод |
|---|---|---|
| Генерация QR | `/api/v2/generate/qr` | POST |
| Статус транзакции по QR | `/api/v1/get-qr-transaction-info/{id}` | GET |
| Выписка по QR | `/api/v1/get-qr-operation-statement` | GET |
| Остатки по счетам | `/api/v1/get-account-balances` | GET |

## Callback (real-time уведомления)

Оптима POST'ит на наш сервер при каждой оплате:
- Наш URL: `https://наш_домен/api/v1/callback`
- Аутентификация входящего: Basic Auth (мы даём банку логин+пароль)

```json
{
  "status": "PROCESSED",
  "sum": 18000.00,
  "note": "...",
  "transactionProcessedDateTime": "2026-06-13T10:00:00Z",
  "transactionId": "abc123",
  "payerClientType": ""
}
```

**Важно:** callback приходит мгновенно. D+1 — только если работать через API выписки (не callback).

## Наша модель интеграции

Работаем через **PIN**, не QR:
1. Родитель вводит PIN ребёнка в приложении → платит
2. Оптима шлёт callback на наш сервер
3. Мы по идентификатору в callback находим студента → помечаем оплату

PIN формат: `ДДММГГГН` (день+месяц+год рождения+порядковый номер)
PIN-список: 84 активных ученика — передан в Оптиму (~25 мая)
