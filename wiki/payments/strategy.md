# Стратегия оплат — Жемчужина

> Обновлено: 2026-06-14

## Ключевое решение: не ждать банки

**Стратегия:** строим систему с ручным вводом платежей сейчас → подключаем автоматику потом.

Причина: банки (Optima, Бакай) ещё не подтвердили автовыгрузку / интеграцию. Ждать — значит стоять. Ручной ввод даёт рабочую систему немедленно.

## Два банка — разные статусы

### Optima Bank

- **Модель:** родитель платит по PIN ребёнка через "Образование" в приложении
- **Бесплатно** для родителя (через приложение)
- **Интеграция:** callback API — мгновенное уведомление
- **Статус:** PIN-список передан (~25 мая), но Аман не подтвердил автовыгрузку
- **Блокер:** неизвестна структура callback для Education/PIN платежей

Подробно → `wiki/payments/optima.md`

### Бакай Bank

- **Подтверждено Айжан 05.06:** есть интеграция под 1С
- **Доступные функции API:** выписка по счёту, QR оплаты с комментарием, остаток на счёте, QR перевода, межбанк/внутрибанк
- **Наш приоритет функций:**
  1. QR оплаты с комментарием → генерировать на каждый инвойс (ФИО ребёнка + месяц)
  2. Выписка по счёту → автоматический матчинг платежей с инвойсами
  3. Остаток на счёте → дашборд собственника

- **Открытые вопросы:**
  - Договор на интеграцию или стандартный API?
  - Sandbox / тестовый стенд?
  - Авторизация: API-ключ или OAuth?

## Очерёдность реализации

1. **Сейчас:** ручной ввод платежей (модуль Дети + Оплаты)
2. **После:** Optima callback — если Аман подтвердит структуру callback
3. **Параллельно или после:** Бакай QR + выписка — после получения API доступа
4. **В конце:** автоматический матчинг, дашборд, уведомления родителям
