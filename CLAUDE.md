# Инструкции для Claude — проект Жемчужина

## Триггер "старт"

Когда пишут **"старт"** — выполнить без лишних слов:
1. Прочитать `context/summary.md` — точка входа
2. Прочитать свежий лог группы: `logs/YYYY-MM-DD.md` (сегодня и вчера)
3. Выдать брифинг: где мы, что открыто, что появилось нового в группе

---

## Главное правило: сначала понимание — потом реализация

1. Изложи как понял задачу
2. Задай уточняющие вопросы если нужно
3. Если видишь лучший подход — скажи и объясни
4. Получи подтверждение → только потом реализация

Простые однозначные правки — можно сразу.

## Принципы работы

### Действую автономно:
- Читаю файлы, логи, документы проекта
- Сохраняю информацию в папку проекта
- Обновляю существующие документы когда появляются новые данные
- Структурирую и форматирую документы

### Сначала спрашиваю:
- Создаю новые разделы или папки
- Переименовываю или удаляю файлы
- Значительные правки в утверждённых документах
- Формулирую сообщения для группы или команды

### Никогда без явного запроса:
- Не пишу в группу Telegram
- Не отправляю ничего никуда

## Структура папки

```
D:/aispace/Pearl/
├── CLAUDE.md                  ← правила работы
├── context/
│   ├── summary.md             ← ТОЧКА ВХОДА: читать первым
│   └── project_brief.md      ← детальный контекст проекта
├── raw/                       ← сырые расшифровки голосовых
├── analysis/                  ← анализ, вопросы, структурированные данные
├── logs/                      ← логи группы Telegram (по дням)
├── logs/terminal/             ← сессионные логи Claude
└── media/                     ← фото, голосовые
```

### Правило загрузки контекста
1. Читай `context/summary.md` — точка входа
2. Нужно больше деталей → `context/project_brief.md`
3. Детали по теме → `analysis/` → `raw/`
4. История по дням → `logs/terminal/YYYY-MM-DD.md`
